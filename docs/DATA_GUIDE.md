# Справочник по экспорту Samsung Health (сон)

> **Быстрый старт** (см. README):
> 1. `uv run samsung-sleep <export.zip> -o output --html` — конвертирует экспорт (zip или
>    распакованную папку) в CSV-таблицы + `llm_context.md` + опциональный HTML-дашборд.
> 2. `uv run streamlit run streamlit_app.py` — интерактивный дашборд с таблицами и графиками.
>
> Для интерпретации в LLM прикладывайте файлы из `output/` (обязательно `llm_context.md` —
> в нём правила против ложных выводов о «днях без сна» из-за снятых часов).

Этот документ описывает структуру стандартного экспорта «Загрузить личные данные» из Samsung Health (директория/архив вида `samsunghealth_<user>_<id>`). Данные носимого устройства (например **Galaxy Watch8**) — основной источник; телефон может фигурировать как источник старых ручных записей и агрегатов. Конкретные значения ниже (`deviceuuid`, даты, профиль) приведены как пример из одного экспорта — у вас они будут свои; код опирается не на них, а на имена таблиц.

---

## 1. Общие правила чтения (критично)

1. **Формат CSV**: первая строка каждого файла — метаданные (`имя_таблицы,версия,счётчик`), реальный заголовок — вторая строка. Файлы начинаются с BOM (`utf-8-sig`). У многих строк висячая запятая в конце → лишняя пустая колонка.
2. **Все timestamps в CSV — в UTC.** Колонка `time_offset` (здесь всегда `UTC+0300`) хранит локальный пояс — его нужно **прибавлять** для получения местного времени. Пример: сон «18:35–23:41» в файле = реальные 21:35–02:41 по Москве. Без этой поправки вся аналитика времени отхода ко сну будет ошибочной.
3. **Timestamps в JSON-файлах** (`jsons/`) — миллисекунды Unix epoch (тоже UTC).
4. `goal_bed_time` / `goal_wake_up_time` — миллисекунды от полуночи **локального** времени (82800000 = 23:00, 25200000 = 07:00).
5. **Связи между таблицами**: `datauuid` — первичный ключ записи; `sleep_stage.sleep_id` → `sleep.datauuid`; `sleep.combined_id` → `sleep_combined.datauuid`; `sleep_raw_data.sleep_uuid` → `sleep.datauuid`.
6. **Колонки `binning_data` / `custom` / `data`** содержат имя JSON-файла. Сам файл лежит в `jsons/<имя_таблицы>/<первый_символ_uuid>/<имя>.json`.
7. `deviceuuid` различает источники: фильтруйте по `+5J4ES+Jsr` (часы), чтобы отсечь телефонные/ручные записи.

## 2. Ядро данных о сне

### `com.samsung.shealth.sleep` — главная таблица: одна строка = одна сессия сна
104 сессии, 102 с оценкой. Ключевые поля:

| Поле | Смысл |
|---|---|
| `start_time` / `end_time` (префикс `com.samsung.health.sleep.`) | Лёг / встал (UTC!) |
| `sleep_score` | Итоговая оценка сна 0–100 (то, что показывает приложение) |
| `sleep_duration` | Длительность сна в **минутах** |
| `efficiency`, `sleep_efficiency_with_latency` | Эффективность сна, % (доля сна от времени в постели) |
| `sleep_latency` | Время засыпания, **мс** |
| `sleep_cycle` | Число циклов сна |
| `total_rem_duration`, `total_light_duration` | REM и лёгкий сон, минуты (глубокий = duration − rem − light − wake) |
| `physical_recovery`, `mental_recovery` | Физическое/ментальное восстановление, % (из детализации оценки) |
| `movement_awakening` | Двигательная активность/пробуждения, % |
| `*_score` / `*_weight` (deep, rem, wake, latency, total_sleep_time) | Компоненты, из которых складывается `sleep_score` (балл × вес) |
| `sleep_type` | Тип сессии: `5` — обычная автозапись с часов (71 шт.), `-1` — короткие сессии ~3.1 ч (23 шт.), `3` — 8 шт.; точная семантика кодов не документирована |
| `combined_id` | Ссылка на объединённую ночь в `sleep_combined` |
| `goal_bed_time` / `goal_wake_up_time` | Целевой режим, мс от локальной полуночи |
| `factor_01..10` | Внутренние факторы оценки. Проверено по данным: `factor_06` = сон в минутах, `factor_01` ≈ % глубокого, `factor_02` = глубокий в мин, `factor_05` = бодрствование в мин. Остальные не расшифрованы |

### `com.samsung.health.sleep_stage` — гипнограмма
5862 сегмента; одна строка = непрерывный интервал одной стадии. `sleep_id` связывает со сном. Коды `stage` (проверены по распределению длительностей):

- `40001` — бодрствование (9.6% общего времени)
- `40002` — лёгкий сон (53.5%)
- `40003` — глубокий сон (16.5%)
- `40004` — REM (20.4%)

Это **главный файл для построения гипнограммы** и подсчёта фрагментации (число сегментов awake, длина пробуждений).

### `com.samsung.shealth.sleep_combined` — объединённые ночи
Когда сон разбит на несколько сессий (проснулся ночью, часы закрыли сессию), Samsung склеивает их в одну «ночь» с пересчитанной оценкой. 4 строки, на них ссылаются 9 сессий из `sleep`. **Для дневной статистики берите combined-запись, а не отдельные куски**, иначе ночь посчитается дважды.

### `com.samsung.shealth.sleep_raw_data` — сырьё алгоритма
Указатели на JSON в `jsons/com.samsung.shealth.sleep_raw_data/`: поминутные блоки с агрегатами акселерометра (`acc`) и RR-интервалами (`rri_info`, мс между ударами сердца). Это вход алгоритма стадирования — полезно только если захотите свой классификатор стадий или честный расчёт HRV за ночь.

### `com.samsung.shealth.sleep_snoring` — храп
10 записей, `duration` = секунды храпа (у вас везде 0 — храп не зафиксирован либо запись с телефона не велась полноценно). JSON-файлы `custom` пустые.

### `com.samsung.shealth.vitality.nap_data` — дневной сон
Интервалы дневного сна + `score_before`/`score_after` — как нап изменил «Energy score». Удобно для оценки эффекта дневного сна.

## 3. Физиология во время сна (часы пишут это в основном ночью)

Все четыре таблицы устроены одинаково: строка = период записи (обычно ночь), детали — в `binning_data` JSON.

| Файл | Что внутри | JSON-гранулярность |
|---|---|---|
| `com.samsung.health.hrv` | Вариабельность пульса | бины ~30 с: `sdnn`, `rmssd` (мс). **RMSSD — ключевой маркер восстановления** |
| `com.samsung.health.skin_temperature` | Температура кожи запястья; в CSV сразу есть `temperature`/`min`/`max` за ночь | поминутно: `mean/min/max`, °C. Отклонение от базлайна — маркер болезни/перегрева |
| `com.samsung.health.respiratory_rate` | Частота дыхания; `average` в CSV | поминутно, вдохов/мин (0 = нет данных). Норма во сне ~12–18; рост — признак стресса/болезни |
| `com.samsung.health.oxygen_saturation.raw` | Сырой оптический сигнал SpO2 (`channel`) — **не проценты**, для анализа бесполезен |
| `com.samsung.shealth.tracker.oxygen_saturation` | Готовые значения SpO2, % (`...spo2`) — используйте этот файл; min < 90% — повод смотреть на дыхание во сне |
| `com.samsung.shealth.tracker.heart_rate` | Пульс: CSV — отсчёты/агрегаты, JSON `binning_data` — поминутные `heart_rate/min/max`. Минимальный ночной пульс — второй ключевой маркер восстановления |
| `com.samsung.health.movement` | Поминутная интенсивность движения (binning JSON) — косвенно беспокойность сна |

## 5. Служебные файлы (для анализа не нужны)

`device_profile` (список устройств), `user_profile` (профиль: пол, дата рождения, рост, вес — извлекается в `llm_context.md`), `preferences`/`service_preferences` (настройки приложения), `badge`/`best_records` (награды/рекорды), `insight_message` (тексты подсказок приложения), `report` (еженедельные отчёты), `shm_device`, `social.service_status`, `activity_level`, `files/` (фото профиля).

## 6. Шаблон утилитарной интерпретации ночи

Для каждой ночи (combined-запись или одиночная сессия из `sleep`):

1. **Режим**: лёг/встал в локальном времени (UTC + offset), сравнить с `goal_bed_time`/`goal_wake_up_time`.
2. **Количество**: `sleep_duration` (цель 7–9 ч), `sleep_latency` (норма < 30 мин), `efficiency` (норма > 85%).
3. **Архитектура** (из `sleep_stage`): % глубокого (норма 13–23%), % REM (норма 20–25%), число и длина пробуждений, `sleep_cycle` (норма 4–6).
4. **Восстановление**: ночной RMSSD (HRV) и минимальный ночной пульс **в сравнении со своим средним за 2–4 недели** — абсолютные значения малоинформативны, важна динамика.
5. **Красные флаги**: SpO2 min < 90%, частота дыхания заметно выше личной нормы, температура кожи +0.5°C к базлайну.
6. **Контекст**: тренировка/шаги накануне, дневной нап (`nap_data`) — для объяснения отклонений.

### Пример (ночь 9–10 июня 2026, реальные данные)

Лёг 21:35, встал 02:41 (5.1 ч в постели) — **оценка 45/100**: спал всего 306 мин при долгом засыпании (102 мин). При этом архитектура сна хорошая: глубокий 18%, REM 26%, эффективность 95%, 3 цикла. RMSSD ~30 мс, дыхание 11.6/мин, SpO2 95%, пульс мин. 59. Интерпретация: низкий балл — целиком из-за **недостаточной длительности и позднего засыпания**, а не качества; физиология в норме.

## 7. Известные грабли

- Не суммируйте сессии, входящие в `sleep_combined`, вместе с combined-записью — дубль.
- 2 сессии без оценки и без стадий (старая ручная запись с телефона 2025-04-20 и одна без анализа) — отсеивать по пустому `sleep_score`.
- Много коротких сессий (~3 ч, `sleep_type=-1`) — у пользователя фрагментированный/полифазный сон; «ночь» лучше определять группировкой сессий по календарным суткам с учётом локального времени, а не по одной записи.
- `respiratory_rate` бины с 0.0 = отсутствие сигнала, выкидывать из средних.
- `oxygen_saturation.raw` — не проценты; проценты в `tracker.oxygen_saturation`.
